5. Interpretation proposee

La discussion a conduit a une lecture en quatre couches :

Couche Role Exemples
L0 - Sources canoniques Materiaux d'origine, a conserver stables. BibTeX JabRef, PDF, pages web, docs R, Rd, scripts.
L1 - Extractions Representations structurees issues des sources. TEI GROBID, Markdown web, metadonnees BibTeX, tables dataset/package.
L2 - Knowledge Graph Relations entre les entites du projet. Paper, Dataset, Package, Method, Formula, Concept, Evidence.
L3 - Wiki Syntheses lisibles et stabilisees. Pages concept, dataset, software, paper.

6. Place de JabRef

JabRef doit piloter le fichier : 

corpus/bib/references.bib

Son role est bibliographique. Il gere les references, les DOI, les auteurs, les titres, les liens vers les PDF locaux et les priorites. Il ne remplace ni GROBID, ni le KG, ni le wiki.

Exemple de champs utiles dans une entree BibTeX :

@article{muenchow2012_lsl,
  title = {...},
  author = {...},
  year = {2012},
  doi = {...},
  file = {../papers/raw_pdf/spDataLarge_lsl - landslides.pdf},
  keywords = {spDataLarge, lsl, landslide, spatial-cv},
  kg_dataset = {spDataLarge::lsl},
  kg_method = {glm, spatial-cv},
  kg_priority = {high},
  kg_status = {pdf_downloaded}
}

7. Place de GROBID

GROBID est reserve aux PDF scientifiques : 

corpus/papers/raw_pdf/*.pdf
        |
        v
corpus/papers/tei/*.tei.xml

Il sert a extraire la structure des articles : titre, auteurs, abstract, sections et references. Pour les livres ou documentations deja disponibles en HTML ou Markdown, il est preferable de les convertir directement en Markdown et de les placer dans corpus/web_md/.

8. Place du KG et du wiki

Le KG doit repondre a des questions structurees :

  • Quel papier utilise spDataLarge::lsl ?
  • Quel dataset est associe a une formule glm ?
  • Quel package R documente GWR ?
  • Quel papier cite Fotheringham ?
  • Quelle page wiki synthetise MGWR ?
